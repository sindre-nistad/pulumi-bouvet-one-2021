Pulumi

Infrastructure as real code

Which means, we may use the entire ecosystem for a given language

including the possibility of debugging

Pulumi

compiles the code to Terraform (and encrypts secrets*)

Imperative, unlike Terraform's declarative (domain specific) language

Experiences

👍 There is no need to learn a new (configuration) language

👍 You can use existing modules, and libraries

👍 Support for unit tests

👍 Supports many cloud providers

Experiences

👎 Need to learn a new framework

👎 Closely coupled to the different cloud provider's APIs

Which can be alleviated by using @pulumi/cloud

👎 There are few, and sometimes no default values for creating resources

I've used the Azure portal for exploration / figuring out which resources are necessary to create a given resource in Pulumi
